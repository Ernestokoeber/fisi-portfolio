Project Description

PaarKurs is a finance-management app for households and couples. It allows joint capture, evaluation, and categorization of expenses — from receipt scanning through a KPI dashboard to savings tips based on the user's own spending patterns.

Classic frontend/backend split: a React SPA on the frontend, an asynchronous FastAPI backend, and PostgreSQL plus Redis as the data layer. AI-based receipt OCR via Gemini Vision with Tesseract OCR as an offline fallback. Fully offline-capable through IndexedDB synchronization.

Technical Approach

  • ° React 18 + TypeScript + Vite as the frontend toolchain
  • ° Tailwind CSS for styling, Zustand for state, TanStack Query for server state, Recharts for visualizations
  • ° FastAPI with asynchronous SQLAlchemy as the backend
  • ° PostgreSQL as primary database, Redis for cache and sessions
  • ° Gemini Vision API for AI-based receipt OCR, Tesseract OCR as fallback
  • ° Cloudflare Pages for frontend hosting, Railway for backend and database
  • ° IndexedDB for offline storage with automatic synchronization

Features

  • ° Receipt upload with AI-based text recognition (Gemini Vision with Tesseract fallback)
  • ° Household sharing: both partners see all transactions and can edit them
  • ° Dashboard with KPIs (expenses, savings, top categories, partner activity feed)
  • ° Shopping list, to-do system with priorities, fixed-cost tracker
  • ° Savings tips based on spending analysis, price comparison, and budgeting
  • ° Offline mode with IndexedDB and automatic synchronization

Focus Areas

Architecture: Clean separation between the React SPA frontend (Cloudflare Pages) and the asynchronous FastAPI backend (Railway). PostgreSQL as the source of truth, Redis for cache and sessions, Railway volume for image assets.

AI integration: Receipt recognition runs primarily on Gemini Vision; large images or outages fall back to Tesseract OCR as a deterministic backup. Text recognition stays usable even without a cloud connection.

Offline-first: All input is persisted to IndexedDB and synced to the backend in the background. The user stays operational even without a connection.

Shared state for two users: Household-sharing logic with a partner activity feed, so both partners always see the latest data without notifications or push infrastructure.
